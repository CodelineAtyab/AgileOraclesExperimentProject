 Tiny Ticketing Subsystem

📌 User Story

As a budding Java developer,
I want to create a tiny ticketing subsystem that distinguishes between complaints and feedback,
so that my team-mates and I can practise encapsulation, inheritance, abstract classes and static fields in a real-world context.

🚀 Additional Implementation Tips

Validate rating in FeedbackTicket constructor (throw IllegalArgumentException if not 1-5).
Mark numberOfTickets as private static int and never expose a setter.
Override toString() in each subclass to simplify list printing (optional but useful).
Use final where appropriate (e.g., for constant strings).
